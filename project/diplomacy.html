OVERVIEW

Diplomacy is a strategic, turned based game that pits players against one another. Players must navigate the tumultuous game map in order to conquer territories and gain a strategic advantage. Diplomacy is ultimately a game of alliances, negotiations,and betrayal. In order to succeed, you must stand alone, but also know how to pick your enemies and allies. Only one player emerges victorious, after having conquered their former friends and enemies. For full documentation and source code, please visit github.

INTRODUCTION

The Diplomacy Game is played by seven players which represent the seven "Great Powers of Europe" in the years prior to World War I. Each of Great Power is subdivided into provinces. There are three types: inland, water, and coastal. Only Armies can move on inland provinces and only Fleets are allowed to move on water provinces. There are a total of 34 inland and coastal provinces on the game board and they are designated as supply centers. This game app is part of my software development and framework class project. The project took the whole semester to complete [approximate 14 weeks]. This gaming project was built in reference to the online Dimplomacy desktop application Backstabrr.

DESIGN PROCESS

  1. Prototyping:

    Our group started the game app prototype by brainstorming the list of features we wanted in the game. Most features were referenced to the online Backstabbr and the orginal Diplomacy game documentation which was given by the professor. After deciding which features should be on the game app, then we had divided the workload. Since one team member and I have had more experience in web design, we were both responsible for the prototype design which was demoed during the fourth week of the semester. The rest of the group started to focus on the backend logic. We started making wireframe using Bootstrap framework and HTML/CSS. The UI design took us about 1 week and got my group one of the top two highest scores during the prototype demo. This interface was good enough for no further modification during the project developement. We spent the rest of the semester focusing on the backend logic of the game.

  2. The Back-end Design

    3. The backend logic was very difficult. All of us has very minimal to none knowledge of JavaScript. We all started learning JavaScript from stratch by ourself using the available resource on the web. One of our most concerned feature was hosting the game. We first thought of using the database, but we did not make much progress. After days of research, we had decided to do the hosting feature through wifi connection. That means, two or more players can join the same game lobby only if they are connecting to the same wifi network. The game full instruction (logic) has not been finished by the project deadline. We have spent most of time on game host and map interaction.

  3. Interactive Map:

    4. screenshots of Map

    Another part of the project that took us long to do was the map interaction. We were given a PNG file of the world war I map image. To make an interactive map, we need to convert this png file to a svg file. My group used inkscape software to convert this png to svg file. For the exact location of each province, we had to trace each province's border by border, it also took a quiet amount of our project time. After each border was traced, we were able to implement the map click feature.
